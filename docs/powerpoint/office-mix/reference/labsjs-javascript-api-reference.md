
# LabsJS JavaScript-API-Referenz
Übersicht des LabsJS JavaScript-Objektmodell

 _**Gilt für:** apps for Office | Office Add-ins | Office Mix | PowerPoint_

Die LabsJS -Referenzdokumente der [TypeScript](http://www.typescriptlang.org/)-Datei,  **labs-1.0.42.d.ts**, sortiert das LabsJS-Objektmodell in Module.

## LabsJS-Objektmodell

Das LabsJS-Objektmodell ist in fünf Module unterteilt:


- [LabsJS.Labs](../../reference/office-mix/labsjs.labs.md). Das Labs-Modul enthält einen Satz von wichtigen APIs, mit denen die Labs erstellt werden können. Sie bieten einen Einstiegspunkt für die Lab-Entwicklung.
    
- [LabsJS.Labs.Core](../../reference/office-mix/labsjs.labs.core.md). Die Kernschnittstellen, Datenstrukturen und Klassen, die von LabsJS und dem Präsentationstreiber freigegeben werden (in diesem Fall Office Mix), um eine Brücke zwischen den beiden zu schaffen.
    
- [LabsJS.Labs.Core.Actions](../../reference/office-mix/labsjs.labs.core.actions.md). Diese APIs stellen die Vorgänge dar, die ein Lab mit dem aktuellen Verhalten angibt, und sie sind nützlich Für Entwickler, die neue Komponenten erstellen (andere als die Standard-Komponenten), oder Verbindungen mit einem neuen Treiber entwickeln (andere als Office Mix).
    
- [LabsJS.Labs.Core.GetActions](../../reference/office-mix/labsjs.labs.core.getactions.md). Mit diesen APIs können Sie Aktionen abfragen, die zuvor auf dem Server auftraten.
    
- [LabsJS.Labs.Components](../../reference/office-mix/labsjs.labs.components.md). Diese APIs stellen die vier Standard-Komponenten dar, die aktuell in Labs verfügbar sind (Aktivität, Auswahl, Eingabe und Dynamisch).
    
Jedes Modul enthält eine Gruppe von Elementen, die aus einem oder mehren der folgenden Elementtypen bestehen:


- Klassen
    
- Schnittstellen
    
- Funktionen
    
- Enumerationen
    
- Variablen
